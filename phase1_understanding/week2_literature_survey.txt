RESEARCH PAPERS REVIEWED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Paper 1: "Compiler Error Messages Considered Unhelpful: The Landscape 
          of Text-based Programming Error Messages"
  Authors : Denny, P., Luxton-Reilly, A., Tempero, E., Lin, J.
  Year    : 2014
  Key Finding:
    - Novice programmers misinterpret GCC error messages approximately
      90% of the time on first reading
    - The most confusing errors involve the words "token", "lvalue",
      and "declaration"
    - Proposed human-written substitutions for 10 most common errors
  Relevance: DIRECT — supports the need for this project

Paper 2: "The Design of Elm: A Friendly Functional Language for Beginners"
  Author  : Evan Czaplicki
  Year    : 2012 (Language), 2016 (Error Systems Paper)
  Key Finding:
    - Elm redesigned its compiler to output long, conversational error
      messages in plain English
    - Students reported significantly reduced debugging time
    - Elm errors explain WHAT went wrong, WHY, and HOW to fix it
  Relevance: HIGH — design inspiration for this project's output format

Paper 3: "Neural Program Repair" 
  Authors : Chen et al.
  Year    : 2019
  Key Finding:
    - Used sequence-to-sequence deep learning models to suggest 
      source code repairs automatically
    - Did NOT focus on natural language explanation for students
    - Requires large training datasets and GPU compute
  Relevance: MEDIUM — shows AI can suggest fixes; too heavy for this project

Paper 4: "Automated Program Repair Using Execution Results and Text"
  Authors : Mechtaev et al.
  Year    : 2018
  Key Finding:
    - Combined static analysis with text-based descriptions of bugs
    - Useful for synthesis of fix templates
  Relevance: LOW — focuses on automated repair, not explanation

Paper 5: "DeepFix: Fixing Common C Language Errors by Deep Learning"
  Authors : Gupta, R., Pal, S., Kanade, A., & Shevade, S.
  Year    : 2017 (AAAI Conference)
  Key Finding:
    - Used Recurrent Neural Networks (RNNs) to automatically predict and 
      fix syntax and compilation errors in C programs submitted by students.
    - Demonstrated that deep learning can capture the syntax of C language
      sufficiently to locate and repair errors.
  Relevance: VERY HIGH — Serves as a foundational AI approach for tackling C 
             compiler errors, similar to our AI Tutor's goal, but our project 
             focuses more on the *explanation* aspect alongside the fix.

Paper 6: "Transformer Models / Large Language Models for Code Repair"
  Authors : (General literature trend, e.g., CodeBERT by Feng et al., 2020)
  Year    : 2020-Present
  Key Finding:
    - Transformer-based architectures (like Flan-T5, used in this project)
      outperform traditional Seq2Seq models in understanding coding context.
    - They can generate high-quality natural language explanations for code snippets.
  Relevance: HIGH — Directly justifies the project's choice of using a 
             Transformer model (Flan-T5) for rewriting error messages.

TOOLS AND SYSTEMS REVIEWED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Tool 1: GCC (GNU Compiler Collection)
  Strengths  : Standard, widely available, detailed error locations
  Weaknesses : Technical jargon, no fix suggestions, no plain English
  Status     : Baseline — this project builds ON TOP of GCC

Tool 2: Clang Compiler (LLVM Project)
  Strengths  : Better error messages than GCC, shows caret (^) markers
               to point to the exact character causing the error
  Weaknesses : Still uses technical language; no fix recommendation
  Example    : clang shows "did you mean 'printf'?" — slight improvement
  Status     : Closer to user-friendly but still not for beginners

Tool 3: Python Tutor / Thonny IDE
  Strengths  : Beginner-friendly error display for Python programs
               Shows a turtle animation of code execution
  Weaknesses : Python only — NO equivalent exists for C language
  Status     : Confirms demand for beginner-friendly tools; gap in C

Tool 4: ChatGPT / GitHub Copilot
  Strengths  : Can explain any error in plain English, suggests fixes
  Weaknesses : Requires internet connection, general-purpose (not C-specific),
               may suggest insecure or incorrect fixes, no GCC integration
  Status     : General solution; this project provides offline C-specific version

Tool 5: Dr. Explain (online GCC explainer websites)
  Strengths  : Some websites translate GCC errors manually
  Weaknesses : Not integrated with compiler, must copy-paste error,
               limited error coverage, no secure fix validation

CONCLUSION OF LITERATURE SURVEY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Research confirms that beginner-unfriendly compiler errors are a documented
and significant problem in computing education. Existing tools either:
  (a) Are language-specific (Python only) and not available for C
  (b) Require internet access and are general-purpose (ChatGPT)
  (c) Partially improve messages but still use technical jargon (Clang)
  (d) Require significant compute resources (Neural Program Repair)

This project fills the gap by providing an offline, C-specific, 
lightweight, rule-based + AI-assisted error message rewriter.
