ABSTRACT
========
GCC compiler error messages are technical, terse, and difficult for 
beginner C programmers to understand. This project presents an AI-based 
tool, the "Compiler AI Tutor", that automatically rewrites GCC error 
messages into plain English explanations with secure fix suggestions. 
A rule-based matching engine is used as the primary approach, backed 
by an annotated dataset of 10 representative C compiler errors. The system 
integrates directly with GCC via Python subprocess calls and outputs a 
structured, beginner-friendly terminal interface. Testing on all 10 error 
types demonstrates 100% detection accuracy, 100% secure fix rate, and 
an average clarity improvement of +222% over raw GCC output.

CHAPTER 1: INTRODUCTION (Page 1)
==================================

1.1 Background and Motivation
  - GCC produces cryptic error messages that beginners cannot interpret
  - Study by Denny et al. (2014): 90% of students misread GCC errors
  - Existing tools (Clang, ChatGPT) are incomplete or need internet
  - This project addresses the gap: offline, C-specific, secure tutor

1.2 Problem Statement
  GCC error messages like "expected ';' before 'printf'" provide no
  explanation, no context, and no fix. Beginners are left confused and
  resort to trial-and-error—often adopting insecure coding habits.

1.3 Objectives
  Obj 1: Collect and classify common GCC errors for C programs
  Obj 2: Create an annotated dataset of errors with explanations and fixes
  Obj 3: Build a rule-based AI module to rewrite errors in plain English
  Obj 4: Integrate the module with GCC via Python subprocess
  Obj 5: Ensure all fix suggestions are secure (no gets, strcpy, blind casts)
  Obj 6: Evaluate the improvement in error message clarity

1.4 Scope and Limitations
  In Scope  : C language, GCC, 10 representative error types, offline use
  Out of Scope: Runtime errors, other languages, GUI, cloud integration

CHAPTER 2: LITERATURE SURVEY (Page 5)
=======================================

2.1 Compiler Error Research
  - Denny et al. (2014): Novices misinterpret GCC errors 90% of the time
  - Traver (2000): Proposed categorization of compiler error usability
  - Key insight: Errors need BOTH a plain explanation AND a correct fix

2.2 Friendly Compiler Design
  - Elm Language (Czaplicki, 2016): Conversational, friendly error messages
  - Lesson: Tone + explanation matters as much as technical accuracy
  - Java compiler: Type errors explained with context

2.3 AI-Based Code Tools
  - Neural Program Repair (Chen et al., 2019): seq2seq for fix suggestion
  - GitHub Copilot, ChatGPT: general-purpose, internet-required
  - Gap: No C-specific, offline, security-aware tool exists

2.4 Gap Identified
  No existing solution combines: C-specific + offline + secure fixes +
  GCC integration + plain English. This project fills that gap.
