For a 5-minute demo, show in this order:

1. Show project folder structure — explain phases match weeks
   → Open File Explorer showing phase1 through phase7 folders

2. Open errors/err_001_missing_semicolon.c
   → "This is an intentional bug — look at line 8, no semicolon"

3. Run GCC directly first:
   > gcc -Wall errors/err_001_missing_semicolon.c
   → Show the cryptic error message students would normally see

4. Run AI Tutor:
   > python phase5_ai_rewriting/rewriter.py errors/err_001_missing_semicolon.c
   → Show the friendly, structured, explained output

5. Run the full demo:
   > phase7_final\run_all_tests.bat
   → Watch all 10 errors be processed and explained

6. Open annotated_errors.json — show the dataset structure
7. Open rewriter.py — show the 6-module pipeline briefly

Talk points:
  - "This works offline — no internet"
  - "All fixes are security-validated — no gets(), no blind casts"
  - "Adding a new error is easy — just add a JSON entry"
