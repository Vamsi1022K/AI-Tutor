COMPARISON 1 — Missing Semicolon
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
BEFORE (GCC):
  main.c:8:5: error: expected ';' before 'printf'

AFTER (AI Tutor):
  💡 What it means:
     You forgot to put a semicolon (;) at the end of a statement.
     In C, every statement must end with a semicolon.
  🔧 How to fix it:
     Add a semicolon after the statement:  int x = 5;
  🔒 ✅ Safe Fix

  Why better: Explains the RULE (every statement needs ;), not just WHAT is missing

┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
COMPARISON 2 — Implicit Function Declaration
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
BEFORE (GCC):
  main.c:3:5: warning: implicit declaration of function 'printf'
              [-Wimplicit-function-declaration]

AFTER (AI Tutor):
  💡 What it means:
     You are using printf() but have not told the compiler where to find it.
     In C, built-in functions need a header file to be included.
  🔧 How to fix it:
     Add at the very top of your C file:    #include <stdio.h>
  🔒 ✅ Safe Fix

  Why better: Explains the CONCEPT of header files — not just the error name

┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
COMPARISON 3 — Type Mismatch
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
BEFORE (GCC):
  main.c:5:7: error: incompatible types when assigning to type 'int'
              from type 'char *'

AFTER (AI Tutor):
  💡 What it means:
     You are trying to store a text string into a variable that can only
     hold a number (int). An int stores whole numbers only.
  🔧 How to fix it:
     For text: change 'int x' to 'char *x'
     For a number: assign a number like x = 42;
  🔒 ✅ Safe Fix

  Why better: Avoids confusing "char *" jargon, explains int vs text concept.
  Security: Does NOT suggest (int)"hello" blind cast — explains root cause.

COMPARISON 4 — Void Return Value
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
BEFORE (GCC):
  main.c:6:12: error: return with a value, in function returning void

AFTER (AI Tutor):
  💡 What it means:
     Your function is declared as 'void' — meaning it returns nothing.
     But you wrote 'return 42;' which tries to send back a number.
  🔧 How to fix it:
     Option 1: Use 'return;' with no value
     Option 2: Change 'void greet()' to 'int greet()' if returning a value
  🔒 ✅ Safe Fix

  Why better: Explains WHAT void means, gives TWO valid options for the fix

COMPARISON 5 — Format Specifier Mismatch
┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
BEFORE (GCC):
  main.c:5:14: warning: format '%d' expects argument of type 'int'
               but argument 2 has type 'double' [-Wformat=]

AFTER (AI Tutor):
  💡 What it means:
     You used %d in printf which is for integers, but passed a decimal
     number (double). This can cause incorrect output to be printed.
  🔧 How to fix it:
     Use %f or %lf for decimal numbers:
         Change:  printf("%d\n", x);
         To:      printf("%f\n", x);
  🔒 ✅ Safe Fix

  Why better: Explains ALL format specifiers (%d, %f, %lf) so student
              learns the pattern, not just the fix for this one case.

Security Risk: Students Googling fixes and copying insecure code

Common insecure fixes found online for GCC errors:
  Error: Can't read string → Fix copied: char buf[10]; gets(buf);
         → ❌ gets() causes buffer overflow vulnerability

  Error: Type mismatch     → Fix copied: (int*)somePointer
         → ❌ Blind cast causes undefined behavior

  Error: Format mismatch   → Fix copied: printf("%s", num) instead of %d
         → ❌ Wrong direction — makes it worse

AI Tutor prevents all of these:
  ✅ Never mentions gets() in any fix
  ✅ Never suggests blind casting for type errors — explains root cause
  ✅ Teaches all format specifiers explicitly (%d, %f, %lf)
  ✅ Uses zero-initialization in declarations (int y = 0 vs int y)

Security Impact Score:
  Without AI Tutor: Students may encounter 3/10 insecure patterns
  With AI Tutor   : 0/10 insecure patterns in any fix recommendation
