Each fix recommendation in the dataset was reviewed against the following
security checklist before being approved and marked "secure: true".

Security Checklist (applied to every fix):
  [ ] Does the fix use gets()? — REJECT if yes
  [ ] Does the fix use scanf("%s") without width limit? — REJECT if yes
  [ ] Does the fix use strcpy() without bounds? — REJECT if yes
  [ ] Does the fix suggest a blind pointer cast? — REJECT if yes
  [ ] Does the fixed code compile cleanly with: gcc -Wall -Wextra? — MUST PASS
  [ ] Does the fix introduce any undefined behavior? — REJECT if yes
  [ ] Is the explanation factually correct? — VERIFY manually

Subprocess Security:
  [ ] GCC is invoked with list arguments (not shell=True) — ✅ Implemented
  [ ] No user input is passed directly to shell — ✅ Verified
